## 修复钱智汇网站分类标签链接404问题

### 问题分析
- 文章页脚的分类和标签链接使用绝对路径（`/category/xxx/`、`/tag/xxx/`）
- 这些链接在GitHub Pages项目仓库中会指向根目录，导致404
- 实际上根本没有生成分类和标签汇总页面

### 解决方案（方案B - 完整方案）
1. **修改链接为相对路径**：将分类/标签链接改为 `../category/xxx/index.html` 和 `../tag/xxx/index.html`
2. **生成分类汇总页面**：添加 `render_category_index()` 函数，为每个分类生成 `html/category/{分类名}/index.html`
3. **生成标签汇总页面**：添加 `render_tag_index()` 函数，为每个标签生成 `html/tag/{标签名}/index.html`
4. **去除markdown模块依赖**：创建 `simple_markdown_to_html()` 函数替代 `markdown.markdown()`，解决网络无法安装模块的问题

### 修改的文件
- `convert_md_to_html.py`：
  - 第50-51行：分类/标签链接改为相对路径
  - 新增 `simple_markdown_to_html()` 函数（简易Markdown转HTML）
  - 新增 `render_category_index()` 函数
  - 新增 `render_tag_index()` 函数
  - 修改 `main()` 函数：生成分类页和标签页

## SEO深度优化（2026-06-06 09:35）

### 新增功能
1. **面包屑导航**：所有页面顶部加面包屑（首页无），通过 `render_breadcrumb_html(depth, items)` 统一生成
2. **JSON-LD 结构化数据**：文章页(Article+BreadcrumbList)、首页(WebSite+SearchAction)、分类页(CollectionPage)、标签页(BreadcrumbList)
3. **Open Graph + Twitter Card**：所有页面加社交分享元标签
4. **上一篇/下一篇导航**：文章页底部通过 `render_prev_next()` 生成，rel="prev/next"
5. **sitemap.xml 升级**：含 lastmod/changefreq/priority；文章0.8，分类0.6，标签0.4
6. **CSS 新增**：`.breadcrumb`、`.post-prev-next` 样式

### 技术关键点
- 文章 HTML 生成必须在 articles 排序之后（才能知道上下篇顺序）
- `generate_sitemap()` 用 `urllib.parse.quote()` 处理中文路径
- `render_nav(depth)` 通过 depth 统一控制所有页面的相对路径前缀
